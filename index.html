Selected Projects & Research

Things I built to answer a question. Real numbers, and the tradeoffs I would revisit.

Universal NIDS - Hybrid Intrusion Detection

Suricata 7.0 + Zeek logs feeding a CatBoost classifier, with a signature-anomaly fusion layer on top. Soaked it for 6 hours straight on 87,432 flows. Not a single false alert. The IDS literature is full of papers that don't reproduce - I wanted one that did.

  • Stack: Python, FastAPI, Suricata 7.0, Zeek, CatBoost, Pandas, Docker
  • Detection: signature + anomaly + supervised ML, fusion layer for confirmation
  • Validation: 6h soak / 87,432 flows / 0 false alerts · 79% test coverage
  • Tradeoff: chose accuracy over latency; p99 inference is 340 ms, which I am working to bring down.
View NIDS case study

Detection Library - Sigma, SPL, Suricata, Zeek

The rules behind the NIDS pipeline, written out as code. Five Sigma rules + two Splunk hunts + a Suricata UA pack + a Zeek enrichment script - each one tagged with the engine it feeds (signature / anomaly / supervised / unsupervised / fusion) and the MITRE ATT&CK technique it catches.

  • Sigma: PowerShell b64, DNS exfil, C2 beacon, auth burst, SYN scan
  • SPL: beacon stddev, lateral movement chain
  • Sensor: Suricata UA anomaly rules + Zeek conn enrichment
  • Tuning: each rule documents the lever to adjust before deploying
View detection library

Vector - Local-first AI Security Platform

Local-first because I don't want telemetry leaving the box. FastAPI runtime (chosen over Flask for its middleware model), multi-provider router falls across Ollama → Gemini → OpenAI → Grok → NVIDIA. Bounded agents so the model can't reach beyond the tools I've defined for it.

  • Stack: Python, FastAPI, Ollama, sqlite, Windows DPAPI for creds
  • Architecture: local-first, agent-bounded, telemetry stays on-host
  • Status: active dev. Private repo. My team uses it daily.

Long-term commercial project - case study on request.
